<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user keyboard input → modal focus state | Tab and Shift+Tab must not escape the visible modal while it is open |
| DOM removal → restore-focus path | The opener can disappear before close, so restore-focus must check document membership |
| form validation state → accessibility tree | Invalid state and error text must stay aligned with the current input value |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | Settings modal keyboard focus loop | mitigate | Filter focusable elements by visibility so hidden controls do not steal focus or mislead keyboard users |
| T-03-05 | Denial of Service | Restore-focus to a detached element | mitigate | Check `document.contains(previousFocus)` before restoring focus on modal close |
| T-03-06 | Information Disclosure | Validation errors that do not reach assistive tech | mitigate | Keep the error text and invalid state synchronized with the current input value |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs or build-step changes are part of this phase |
</threat_model>
