<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keyboard user → DOM focus order | Skip link, dialog focusability, and visible focus style determine whether keyboard users can navigate safely |
| assistive tech → static HTML | Screen readers consume the live regions, dialog semantics, and labels from the initial DOM |
| CSS → accessibility tree | Screen-reader-only utility and focus-visible rules affect perceivability without altering app logic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Missing live-region semantics for chat history and status | mitigate | Add static log semantics and permanent live regions in index.html so AT can track the UI state without script timing issues |
| T-03-02 | Spoofing | Icon-only controls with ambiguous names | mitigate | Give each control a stable accessible name in index.html so the announced action matches the visible control |
| T-03-03 | Tampering | Keyboard navigation that skips hidden or unreachable content | mitigate | Add skip-link routing and visible :focus-visible styling; keep dialog semantics intact so focus order stays predictable |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs or build-step changes are part of this phase |
</threat_model>
