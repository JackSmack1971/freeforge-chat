<objective>
Add the inline edit entry surface for user-authored messages without changing the resend pipeline yet.

Purpose: clicking a user bubble should swap that bubble for an in-place textarea that is prefilled with the original text, so the edit interaction is discoverable and reviewable before the resend logic lands.

Output: user-message bubbles become editable in place, with cancel/back-out behavior and styling, while all message data still comes from `S.messages`.
</objective>

## Phase Goal

**As a** user reading the chat, **I want to** click one of my own messages and edit it in place, **so that** I can correct a prompt without leaving the conversation view.

| document click handling → message rendering | User interaction selects the edit target |
| transient edit state → message list rerender | Bubble/editor swap must not mutate the underlying conversation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-01 | Tampering | Inline edit entry state drift | mitigate | Keep edit state transient and derive it from `S.messages` plus the active DOM target |
| T-03-02 | Denial of service | Editing while streaming | mitigate | Block edit entry while `S.streaming` is true |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — UI-only edit |
</threat_model>
