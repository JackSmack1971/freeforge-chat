<objective>
Make inline edit confirmation truncate the conversation at the edited user message and resend the edited prompt through the existing `sendMessage()` path.

Purpose: the edit flow should behave like a targeted regenerate from any user message, not a new message-history pipeline.

Output: confirming an inline edit drops the edited message and everything after it, then reuses `sendMessage()` to stream the replacement turn.
</objective>

## Phase Goal

**As a** user confirming an inline edit, **I want to** resend the conversation from the edited message forward, **so that** the assistant response reflects the corrected prompt without maintaining a separate edit history.
