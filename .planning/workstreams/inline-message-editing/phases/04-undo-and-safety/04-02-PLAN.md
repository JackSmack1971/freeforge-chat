## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test harness globals -> app bootstrap | The regression must reproduce the real app bootstrap path, not a bespoke test-only path |
| fake browser storage -> undo state assertions | The test must prove the undo slot stays in memory and never leaks into storage |
| stale undo callback -> restored conversation slice | The test must prove the tokenized slot prevents replaying the wrong edit |
| controlled fake timer -> expiry cleanup | The regression must prove the 6-second window clears deterministically without a real sleep |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | Stale undo callback or timer | mitigate | Assert that a later confirmed edit replaces the active slot, that only the current token can restore, and that the timer-driven expiry clears the slot |
| T-04-05 | Information Disclosure | Browser storage boundary | mitigate | Seed fake `localStorage` and `sessionStorage`, then assert the undo flow never writes either backend |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No installs; use the existing node:test harness only |
</threat_model>
