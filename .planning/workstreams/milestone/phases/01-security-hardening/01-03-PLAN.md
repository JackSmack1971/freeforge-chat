      export function getStoredKey() {
        try { return localStorage.getItem('ff_key'); } catch { return null; }
      }
      export function setStoredKey(key) {
        try { localStorage.setItem('ff_key', key); } catch {}
      }

    CRITICAL CONSTRAINTS:
    - Both functions call localStorage.getItem/setItem directly. Do NOT use LS.get or LS.set.
      LS.get calls JSON.parse (returns null for raw strings); LS.set calls JSON.stringify (double-
      encodes: stores '"sk-or-v1-abc"' instead of 'sk-or-v1-abc', breaking all API calls with 401).
    - These are top-level named exports identical in style to maskKey and fmtCtx already in state.js.
    - They are NOT methods on the LS object.
    - getStoredKey has no parameters.
    - setStoredKey takes exactly one parameter named key.
    - Both are wrapped in try/catch (same pattern as LS methods).
    - Do not add a blank line between the two function declarations (they are a logical pair).
    - Separate this block from its neighbors with one blank line above and one below.

    Do not change any other part of state.js.
  </action>
