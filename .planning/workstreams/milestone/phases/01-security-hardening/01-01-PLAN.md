<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CDN → browser | jsDelivr serves script bytes; browser validates SRI hash before execution |
| browser → openrouter.ai | All fetch() calls cross this boundary; CSP connect-src restricts it |
| LLM output → DOM | Markdown rendering pipeline; DOMPurify sanitizes before innerHTML injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | CDN-served marked@9.1.6 (CVE-patched in v18) | mitigate | Upgrade to marked@18.0.4 with SHA-384 SRI hash; browser blocks execution on mismatch |
| T-01-02 | Tampering | CDN-served DOMPurify@3.1.6 (CVE-2025-26791) | mitigate | Upgrade to DOMPurify@3.4.8 with SHA-384 SRI hash; browser blocks execution on mismatch |
| T-01-03 | Tampering | freeforge.html stale entry point | mitigate | Delete freeforge.html; all traffic to freeforge/index.html only |
| T-01-04 | Information Disclosure | Unrestricted resource loading (no CSP) | mitigate | Add meta CSP: default-src 'none'; connect-src https://openrouter.ai only |
| T-01-05 | Tampering | Unauthorized script injection via missing CSP | mitigate | CSP script-src 'self' cdn.tailwindcss.com cdn.jsdelivr.net; blocks inline/eval/other origins |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — CDN upgrades only (no npm install step) |
</threat_model>
